# 从零搭建 AI 知识库 — 提示词指南

> 不需要任何文件，只需要一段提示词，就能让任何 AI 变成"会记笔记的助手"。

---

## 第一步：创建知识库文件夹

在你想放的位置创建一个文件夹，比如：
- Windows: `D:\MyKnowledge\`
- Mac/Linux: `~/my-knowledge/`

---

## 第二步：给 AI 发这段提示词

复制以下内容，发给你正在使用的 AI（ChatGPT / Cursor / Claude / Copilot 等）：

```
请帮我创建一个个人知识库。要求：

## 知识库位置
（替换为你的文件夹路径）

## 文件结构
在知识库目录下创建以下文件：

1. `_index.md` — 索引文件，格式：
   ```
   # 知识库索引 & 自评
   > 🔴 只是知道概念  🟡 理解实操过  🟢 能独立解决

   ## 分类名 (文件路径)
   | 条目 | 记忆点 | 掌握 | 上次复习 |
   |------|--------|------|----------|
   ```

2. `README.md` — 使用规则

3. 按你的知识领域创建分类文件，比如：
   - `tech/basics.md` — 技术基础
   - `tech/tools.md` — 工具使用
   - `life/thinking.md` — 思维认知

## 知识条目格式
每条知识必须包含：
- **记忆点：** 一句话口诀，像"等它说好了再读"而不是"工作原理"
- **掌握程度：** 🔴只是知道 / 🟡理解实操过 / 🟢能独立解决
- **场景：** 什么情况下遇到的
- **原理：** 通俗解释，专有名词保留但紧跟解释
- **代码：** 项目路径/文件:行号（如果有）
- **关联：** [相关条目](文件.md#标题)（必须用可点击链接）
- **来源：** 项目名，日期

## 你的职责
从现在开始：
1. 对话中遇到重要知识，主动用通俗语言解释并记录到对应文件
2. 用户说"记一下"、"这个重要"时立即记录
3. 踩坑经验记录原因和方法
4. 记录前检查是否已存在，避免重复
5. 新增条目时同步更新 `_index.md`（条目名 + 记忆点 + 掌握 + 上次复习）
6. 不定时提醒用户复习，扫记忆点列，提示更新日期和掌握程度
7. 定期提醒备份知识库
8. 不确定的标"（待验证）"，绝不编造
```

---

## 第三步：确认文件创建

AI 会帮你创建好知识库文件。检查一下：
- [ ] `_index.md` 存在，格式正确
- [ ] 分类文件存在（比如 `tech/basics.md`）
- [ ] `README.md` 存在

---

## 第四步：开始使用

正常和 AI 对话就行。

**自动记录：** 当对话中出现重要知识时，AI 会自动解释并存到对应文件。

**手动触发：** 说以下任意一句：
- "记一下"
- "这个重要"
- "存到知识库"
- "帮我记录这个"

**复习：** 打开 `_index.md`，扫记忆点列。想不起来的点链接看详情，更新"上次复习"日期。

---

## 持久化（可选）

上面的提示词每次新对话都要重新发。如果你用的工具支持持久化规则：

| 工具 | 怎么做 |
|------|--------|
| **Claude Code** | 把提示词内容写入 `~/.claude/CLAUDE.md` |
| **Cursor** | 项目根目录放 `.cursorrules` 文件，内容粘贴提示词 |
| **ChatGPT** | Settings → Custom Instructions → 粘贴提示词 |
| **Copilot** | 项目放 `.github/copilot-instructions.md`，粘贴提示词 |

这样每次新对话自动生效，不用重复发。
