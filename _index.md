## 嵌入式基础 (tech/embedded/basics.md)

| 条目 | 记忆点 | 掌握 | 上次复习 |
|------|--------|------|----------|
| GPIO 速度 vs 传感器时序 | MCU 太快，传感器跟不上，加延时就行 | 🟡 | - |
| 传感器预热与采样平均 | 上电等几秒再校准，多读几次取平均就稳了 | 🟡 | - |
| 传感器校准的两种误差 | 偏移减固定值，比例除系数，两个都有就解方程 | 🟡 | - |
| STM32 printf 不支持 %f | 浮点打印不出来，拆成整数拼小数点 | 🟡 | - |
| 硬件抽象层（HAL） | 踩油门不用管是汽油机还是电动机——接口不变，实现随便换 | 🟡 | - |
| 电机堵转电流 | 卡住不动时电流最大，供电不够就烧或重启 | 🟡 | - |
| ST-Link 必须接 3.3V 才能通信 | ST-Link 的 3.3V 不是供电，是"感知"目标在不在 | 🟡 | - |
| PWM 输出必须配 GPIO 复用 | 定时器配好了但没输出，多半是 GPIO 没设成复用模式 | 🟡 | - |
| 主循环永远不能阻塞 | 嵌入式主循环一阻塞，按钮失灵、舵机卡死、屏幕冻结，全线崩盘 | 🟡 | - |
| 用时间戳代替 delay | delay 是"死等"，millis() 是"看表"——系统停不停摆就差在这 | 🟡 | - |
| 传感器采样要与业务逻辑解耦 | 传感器自己按节奏采样，业务逻辑按节奏读值，谁也不等谁 | 🟡 | - |
| 防抖要考虑采样频率 | 防抖窗口比采样间隔还短，那防抖就是摆设 | 🟡 | - |
| 先定义产品需求再选传感器 | 先想清楚"要解决什么问题"，再决定"用什么零件" | 🔴 | - |
