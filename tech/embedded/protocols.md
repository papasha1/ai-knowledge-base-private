## I2C

**记忆点：** 两根线挂一堆设备，每个有地址，喊谁谁答应。

**掌握程度：** 🟡 用过库，未深入底层

**场景：** OLED（地址 0x3C）和 VL53L0X（地址 0x29）共享 PB6/PB7 两根线，互不干扰。

**原理：** I2C 是"总线"协议，多设备挂在同一对线上，每个设备有唯一 7 位地址。主机发地址，只有对应设备响应。

**两根线：** SCL（时钟线，主机控制节奏）+ SDA（数据线，传 0 和 1）。

**局限：** 速度慢（标准 100kHz，快速 400kHz），长距离易受干扰，需上拉电阻（4.7KΩ）。

**类比：** 一条街上很多户人家，邮递员喊"0x3C 的信"，只有 0x3C 那家开门。

**代码：** `FeedBowlForA/lib/hal/hal_i2c.cpp` — I2C 底层封装

**关联：** [VL53L0X](sensors.md#vl53l0xtof-测距)、[OLED](sensors.md#oledssd1306-显示屏)

## PWM 与舵机控制

**记忆点：** 信号线只传"开多久"，电从电源来，不从信号来。

**掌握程度：** 🔴 只是知道概念

**场景：** MG995 舵机通过 PWM 信号控制，信号线接 STM32 的 PB0（TIM3_CH3），电源从外部 5V 接。

**原理：** PWM（脉冲宽度调制）就是快速地开关，通过改变"开"的时间比例来模拟不同电压。舵机的标准 PWM 周期是 20ms（50Hz），脉冲宽度决定行为：
- 1500µs = 中位（360 度舵机：停止）
- \>1500µs = 正转（越宽越快）
- <1500µs = 反转（越窄越快）

**类比：** 就像快速开关水龙头——开的时间长，流出来的水多；开的时间短，流出来的水少。平均效果像调小了水量。

**关联：** [电机堵转电流](basics.md#电机堵转电流)
